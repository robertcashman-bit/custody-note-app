3. CUSTODYNOTE.COM — WHAT HAPPENS WHEN YOU “SEND EMAIL” FROM THE WEBSITE
================================================================================
Page checked: https://custodynote.com/contact

The site does not POST a form to send mail server-side. Instead, support links use
the same mechanism as the desktop app’s Outlook Web integration:

  https://outlook.office.com/mail/deeplink/compose?to=…&cc=&bcc=&subject=…&body=…

Examples present on the Contact page:
- “Open in Outlook Web” (general support):
  to=robertcashman@defencelegalservices.com, empty subject/body — user completes in OWA.
- “Licence enquiry in Outlook Web”:
  subject=Custody Note Licence Enquiry, prefilled To line.

Observed browser behaviour when activating these links:
- Navigation targets Microsoft’s Outlook Web compose endpoint (not custodynote.com).
- Subframes / requests to login.live.com and aadcdn.msftauth.net may appear while
  Microsoft resolves the signed-in account — user must complete sign-in if needed.
- Sending the message is entirely inside Microsoft 365 / Outlook Web; CustodyNote’s
  servers do not receive the email body.

4. DESKTOP APP — “QUICK EMAIL TO OFFICER” / OFFICER TEMPLATES (Electron)
================================================================================
When you choose Open in Outlook Web from the app, the main process builds the same
style of HTTPS compose URL (see lib/outlookWebComposeUrl.js) and opens it once.

Persistent trace on disk (per Windows user profile / app userData):
  email-launch.log

Each append is one JSON line with ts, event, and data. Typical sequence:
  resolve_account_type — account type, skipConfirm, login hint usage
  url_built — host, path, URL length, truncated flag (body trimmed for length)
  url_launched — launchMethod (e.g. shell vs edge-inprivate), composeSignature check

If the user cancels the privacy confirmation dialog:
  cancelled

Subject and body are not written to email-launch.log (privacy — see openOutlookWebEmail.js).

Console (main): [EMAIL] Opening Outlook (account=…, route=…, len=…, truncated=…, mode=…)

5. WHAT YOU SEE WHEN YOU “SEND” (END-TO-end)
================================================================================
Website contact links: browser opens OWA compose → you sign in if required → Send in Outlook.

Desktop app: Electron opens OWA (or Edge InPrivate per settings) with To/Subject/Body
prefilled → you confirm in Microsoft’s UI → Send in Outlook.

Neither path transmits the finished email through custodynote.com; delivery is via
Microsoft 365 / your mailbox provider.
